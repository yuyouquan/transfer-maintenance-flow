> 版本：v1.0
> 更新日期：2026-03-15
> 访问地址：https://transfer-maintenance-flow-repo.vercel.app

## 一、系统概述

转维流程系统是一套项目转维（从在研团队移交到维护团队）的电子化流程管理工具。系统通过标准化的流水线机制，确保转维过程中各项资料齐全、审核到位、交接有序。

### 核心流程

```
项目发起 → 资料录入与AI检查 → 维护审核 → 信息变更
```

每个项目的转维包含 **4个并行角色**：
- **SPM**：项目管理相关资料
- **TPM（测试）**：测试相关资料
- **底软**：底层软件相关资料
- **系统**：系统集成相关资料

另有 **SQA** 角色负责整体质量把关。

### 两个团队

| 团队 | 职责 | 操作阶段 |
|------|------|----------|
| 在研团队 | 录入转维材料和评审要素 | 资料录入与AI检查 |
| 维护团队 | 审核在研团队提交的材料 | 维护审核 |

## 二、用户角色与权限

### 角色说明

| 角色 | 所属团队 | 主要操作 |
|------|----------|----------|
| SPM | 在研/维护 | 录入/审核SPM相关材料 |
| TPM | 在研/维护 | 录入/审核测试相关材料 |
| 底软 | 在研/维护 | 录入/审核底软相关材料 |
| 系统 | 在研/维护 | 录入/审核系统相关材料 |
| SQA | 在研 | 质量把关，关闭流水线 |

### 切换用户（演示功能）

1. 点击页面右上角的用户头像区域
2. 在下拉菜单中选择要切换的用户
3. 页面会自动刷新，显示该用户视角下的数据

> 注意：不同用户看到的操作按钮和数据范围不同，这是角色权限控制的体现。

## 三、工作台

### 3.1 页面布局

工作台是系统的主页面，分为三个区域：

- **顶部统计卡片**：展示全部项目、进行中、已完成、已关闭的数量
- **左侧项目列表**：展示所有转维申请，支持搜索和状态筛选
- **右侧待办面板**：展示当前用户的待办任务

### 3.2 统计卡片筛选

点击任意统计卡片可快速筛选对应状态的项目：

| 卡片 | 筛选效果 |
|------|----------|
| 全部项目 | 显示所有项目（默认） |
| 进行中 | 仅显示状态为"进行中"的项目 |
| 已完成 | 仅显示已完成转维的项目 |
| 已关闭 | 仅显示被关闭的项目 |

当前选中的卡片会有高亮边框标识。

### 3.3 项目列表

每个项目行显示：

| 列 | 说明 |
|----|------|
| 项目名称 | 含首字母图标、发起人、更新日期，点击可进入详情 |
| 流水线进度 | 进度条 + 当前节点标签，直观展示项目进展 |
| 角色进度 | 4个角色（SPM/TPM/底软/系统）的状态标签 |
| 操作 | 根据当前用户权限显示不同按钮 |

### 3.4 操作按钮说明

| 按钮 | 显示条件 | 功能 |
|------|----------|------|
| 详情 | 所有用户 | 查看项目详情页 |
| 录入 | 在研团队成员 + 资料录入阶段 + 该角色有待录入项 | 进入资料录入页面 |
| 评审 | 维护团队成员 + 维护审核阶段 + 该角色有待审核项 | 进入维护审核页面 |
| 关闭 | 见下方权限规则 | 关闭整条流水线 |

**关闭流水线权限规则：**
- 无角色进入审核阶段时 → 仅**发起人（申请人）**可关闭
- 有角色已进入审核阶段 → 仅**SQA**可关闭

### 3.5 关闭流水线操作

1. 点击"关闭"按钮
2. 弹窗中查看各角色评审状态（如已有角色进入审核）
3. 填写关闭原因（必填，最多500字）
4. 点击"确认关闭"

### 3.6 待办面板

- 显示当前用户的待办任务
- 每条待办显示项目名、当前节点、任务类型（录入/评审）
- 点击待办可直接跳转到对应操作页面
- 可通过右上角箭头收起/展开面板

### 3.7 搜索功能

在搜索框中输入项目名称关键字，按回车或点击搜索按钮进行搜索。清空搜索框会自动恢复全部数据。

## 四、项目详情页

### 4.1 进入方式

- 工作台项目列表 → 点击项目名称
- 工作台项目列表 → 点击"详情"按钮

### 4.2 页面内容

#### 流水线进度

顶部显示4个节点的流水线进度图：
- 灰色圆点 = 未开始
- 蓝色圆点 = 进行中
- 绿色圆点 = 已完成
- 红色圆点 = 失败

每个节点下方有4个角色子节点，悬浮可查看角色状态详情。

#### 项目信息

显示项目基础信息：项目名、项目编号、项目负责人、转维负责人、计划评审日期、备注。

#### 团队信息

左右两栏分别展示在研团队和维护团队的成员信息，按角色排序：SPM → TPM → 底软 → 系统 → SQA。

每个成员卡片显示：姓名、角色标签、部门。

#### 转维材料（CheckList）

展示该项目所有的转维材料检查项，包括：
- 序号、类型、检查项名称
- 责任角色、录入责任人、审核责任人
- 交付件链接
- 录入状态、AI检查状态、审核状态

#### 评审要素

展示该项目所有的评审要素，结构与转维材料类似。

#### 操作历史

展示项目的操作记录时间线，包括操作时间、操作人、操作详情。

## 五、资料录入与AI检查

### 5.1 进入方式

- 工作台 → 点击"录入"按钮
- 待办面板 → 点击录入类型的待办

### 5.2 页面布局

- 顶部：返回按钮、页面标题、项目名、当前角色标签
- 流水线进度：展示当前流水线状态
- 固定操作栏：显示当前角色、录入进度、提交审核按钮
- 主内容区：分为"转维材料"和"评审要素"两个标签页

### 5.3 数据筛选规则

- **在研团队成员**：只能看到自己角色对应的录入项
- **被委派人员**（非在研团队）：只能看到被委派给自己的录入项
- **非在研团队成员**不显示"提交审核"按钮

### 5.4 录入操作

1. 在列表中找到需要录入的项目
2. 点击"操作"列的"录入"按钮
3. 弹出录入弹窗，显示检查项详细信息和AI检查规则
4. 在文本框中填写录入内容
5. 可选择"暂存"（保存为草稿）或"保存"（标记为已录入）

### 5.5 AI检查

录入内容保存后，系统会自动触发AI检查：
- 点击AI检查状态列的标签可查看AI检查详情
- AI检查结果分为：通过（绿色）、不通过（红色）、检查中（蓝色）
- AI检查不通过时会显示具体的检查结果说明

### 5.6 委派操作

当需要将某个检查项的录入工作委派给其他人时：

1. 选择需要委派的行（通过左侧复选框）
2. 点击操作栏的"委派"按钮
3. 在弹窗中选择委派人员（单选）
4. 点击"确认委派"

委派后：
- 该项的"资料录入-责任人"列会更新为被委派人
- 显示紫色"已委派"标签
- 被委派人登录后可在工作台看到录入按钮

### 5.7 提交审核

当所有检查项录入完成后：

1. 确认所有项目状态为"已录入"
2. 点击固定操作栏的"提交审核"按钮
3. 系统会将该角色的资料提交至维护审核阶段

> 注意：只有在研团队中对应角色的成员才能看到"提交审核"按钮。

## 六、维护审核

### 6.1 进入方式

- 工作台 → 点击"评审"按钮
- 待办面板 → 点击评审类型的待办

### 6.2 页面布局

- 顶部：返回按钮、页面标题、项目名、当前角色标签
- 流水线进度：展示当前流水线状态
- 固定操作栏：评审角色、负责人、委派按钮、不通过/通过按钮
- 主内容区：分为"转维材料"和"评审要素"两个标签页

### 6.3 逐项审核

每一行的操作列提供：
- **通过**（绿色）：标记该项审核通过
- **拒绝**（红色）：标记该项审核不通过

### 6.4 批量审核

1. 通过左侧复选框选择多个项目
2. 固定操作栏会出现"批量通过"和"批量不通过"按钮
3. 点击对应按钮可一次性处理多个项目

### 6.5 委派操作

维护审核的委派是**角色级别**的，即将当前角色的所有审核任务一次性委派给另一个人：

1. 点击固定操作栏的"委派"按钮
2. 在弹窗中选择委派人员
3. 点击"确认委派"

委派后：
- 所有审核项的"人工审核-责任人"更新为被委派人
- 显示紫色"已委派"标签
- 操作栏的"负责人"名称同步更新并显示"已委派"标签
- 已被委派的人员不会再出现在委派选项中

### 6.6 审核通过

1. 点击固定操作栏的"通过"按钮
2. 系统询问是否需要创建遗留任务
   - **否，直接通过**：直接完成审核
   - **是，创建遗留任务**：填写遗留任务信息

#### 创建遗留任务

遗留任务用于记录虽然审核通过但仍需后续处理的问题：

| 字段 | 说明 |
|------|------|
| 责任人 | 选择负责处理该遗留问题的人员 |
| 部门 | 输入责任人所在部门 |
| 问题描述 | 描述遗留问题的详细内容 |
| 完成时间 | 设置遗留任务的截止日期 |

可通过"新增任务行"添加多个遗留任务。

### 6.7 审核不通过

1. 点击固定操作栏的"不通过"按钮
2. 填写评审意见（必填）
3. 添加Block任务：

| 字段 | 说明 |
|------|------|
| Block点描述 | 描述具体的阻塞问题 |
| 解除措施 | 描述如何解决该问题 |
| 责任人 | 选择负责解决的人员 |
| 部门 | 输入责任人所在部门 |
| 完成时间 | 设置Block任务的截止日期 |

可通过"新增 Block 行"添加多个Block任务。

确认后：
- 流水线回退到"资料录入与AI检查"阶段
- 在研团队需要根据Block任务进行整改
- 整改完成后重新提交审核

## 七、配置中心

### 7.1 进入方式

点击顶部导航栏的"配置中心"。

### 7.2 功能说明

配置中心用于管理转维材料模板和评审要素模板：

#### 转维材料模板管理

管理所有检查项的标准模板，包括：
- 类型（检查项/交接资料）
- 检查项内容
- 责任角色
- 录入角色
- 审核角色
- AI检查规则

支持新增、编辑、删除模板项。

#### 评审要素模板管理

管理所有评审标准的模板，包括：
- 标准名称
- 说明
- 备注
- 责任角色
- 录入角色
- 审核角色
- AI检查规则

支持新增、编辑、删除模板项。

|------|------|

## 九、状态说明

### 流水线节点状态

| 状态 | 颜色 | 说明 |
|------|------|------|
| 未开始 | 灰色 | 该节点尚未启动 |
| 进行中 | 蓝色 | 该节点正在进行 |
| 已完成 | 绿色 | 该节点已完成 |

| 状态 | 颜色 | 说明 |
|------|------|------|

| 暂存 | 橙色 | 已保存草稿，未正式提交 |
| 已录入 | 绿色 | 已完成录入 |

### AI检查状态

| 状态 | 颜色 | 说明 |
|------|------|------|
| - | 灰色 | 未触发检查 |
| 检查中 | 蓝色 | AI正在检查 |

| 状态 | 颜色 | 说明 |
|------|------|------|

| 审核中 | 蓝色 | 正在审核 |
| 通过 | 绿色 | 审核通过 |
| 不通过 | 红色 | 审核不通过，需整改 |

## 十、常见问题

### Q1: 为什么我看不到"录入"按钮？

- 检查当前登录用户是否为该项目的在研团队成员
- 检查项目是否处于"资料录入与AI检查"阶段
- 检查您的角色是否有待录入的项目

### Q2: 为什么我看不到"评审"按钮？

- 检查当前登录用户是否为该项目的维护团队成员
- 检查项目是否处于"维护审核"阶段
- 检查您的角色是否有待审核的项目

### Q3: 如何将录入/审核工作委派给别人？

- 录入页面：选择对应行，点击操作栏的"委派"按钮
- 审核页面：点击操作栏的"委派"按钮（角色级别委派）

### Q4: 审核不通过后，在研团队如何处理？

审核不通过时，维护团队会创建Block任务。在研团队需要：
1. 在工作台查看Block任务详情
2. 根据Block任务描述进行整改
3. 完成整改后重新提交审核

### Q5: 什么是遗留任务？

遗留任务是审核通过时创建的后续跟进事项。虽然审核已通过，但某些问题需要在后续解决。遗留任务不影响流水线推进。

### Q6: 谁可以关闭流水线？

- 如果没有任何角色进入审核阶段 → 仅申请人（发起人）可关闭
- 如果已有角色进入审核阶段 → 仅SQA角色可关闭
