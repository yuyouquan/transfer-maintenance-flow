### 2.2 流水线进度组件（PipelineProgress）

自定义组件，横向展示四大流程节点及角色子状态。

#### 结构

```
[项目发起] ─── [资料录入与AI检查] ─── [维护审核] ─── [信息变更]
                 ● ● ● ●                 ● ● ● ●
                SPM 测 底 系             SPM 测 底 系
```

#### 设计规格

| 元素 | 规格 |
|------|------|
| 节点圆形 | 直径 40px，居中放图标（Ant Design Icon） |
| 节点连接线 | 高度 2px，连接相邻节点 |
| 节点文字 | 14px，居中显示在节点下方，margin-top: 8px |
| 角色圆点 | 直径 10px，水平排列在节点文字下方 |
| 圆点间距 | 6px |
| 圆点到节点文字距离 | 8px |

#### 节点图标映射

| 节点 | 图标 | Ant Design Icon |
|------|------|-----------------|
| 项目发起 | 火箭 | `RocketOutlined` |
| 资料录入与AI检查 | 编辑 | `EditOutlined` |
| 维护审核 | 审核 | `AuditOutlined` |
| 信息变更 | 同步 | `SyncOutlined` |

#### 节点颜色映射

| 状态 | 圆形背景 | 圆形边框 | 图标色 | 连接线色 |
|------|----------|----------|--------|----------|
| 未开始 | #f3f4f6 | #e5e7eb | #9ca3af | #e5e7eb |
| 进行中 | #eff6ff | #2563eb | #2563eb | 左侧 #4338ca，右侧 #e5e7eb |
| 成功 | #f0fdf4 | #16a34a | #16a34a | #16a34a |
| 不通过 | #fef2f2 | #dc2626 | #dc2626 | #dc2626（到当前节点） |

#### 角色圆点规格

| 状态 | 圆点颜色 | 说明 |
|------|----------|------|
| 未开始 | #d1d5db | 灰色 |
| 进行中 | #2563eb | 蓝色 |
| 完成 | #16a34a | 绿色 |
| 被拒绝 | #dc2626 | 红色 |

- 悬浮（Tooltip）显示：`角色名称 - 状态文字`，如 "SPM - 进行中"
- 圆点顺序固定：SPM、测试、底软、系统

### 2.3 角色圆点组件（RoleDot）

独立小组件，复用于流水线和其他需要展示角色状态的场景。

| 属性 | 类型 | 说明 |
|------|------|------|
| role | enum | SPM / TEST / BSW / SYS |
| status | enum | NOT_STARTED / IN_PROGRESS / COMPLETED / REJECTED |
| size | number | 默认 10px |
| showTooltip | boolean | 默认 true |

### 2.4 Modal 规范

#### 通用 Modal 规格

| 属性 | 值 |
|------|-----|

#### Modal 按钮规范

- 主操作按钮（确认/提交）：紫色实心按钮，右侧

- 危险操作按钮（不通过/关闭）：红色实心按钮
- 按钮间距：8px

## 4. 交互流程

### 4.1 录入流程（全链路）

```
用户进入资料录入页
    │
    ├─→ 系统加载当前角色对应的条目数据
    │   （普通用户：仅看到自己负责的条目）
    │   （角色负责人：看到当前角色所有条目）
    │
    ├─→ 用户点击某条目"录入"按钮
    │   │
    │   └─→ 弹出资料录入 Modal
    │       │
    │       ├─→ 填写资料内容（多行文本）
    │       ├─→ 添加交付件（上传文件/添加链接）
    │       │
    │       ├─→ 点击"暂存"
    │       │   └─→ 关闭 Modal
    │       │       └─→ 录入状态更新为"已暂存"（橙色）
    │       │           └─→ 可再次点击录入继续编辑
    │       │
    │       └─→ 点击"确认"
    │           └─→ 关闭 Modal
    │               └─→ 录入状态更新为"已录入"（绿色）
    │                   └─→ [仅转维材料] 触发 AI 智能检查
    │                       │
    │                       ├─→ AI 检查状态变为"检查中"（蓝色 Spin）
    │                       │   └─→ 等待 2-3 秒（Mock）
    │                       │
    │                       ├─→ 检查通过 → 状态变为"通过"（绿色）
    │                       │
    │                       └─→ 检查不通过 → 状态变为"不通过"（红色）
    │                           └─→ 用户可重新点击"录入"修改
    │                               └─→ 重新触发 AI 检查
    │
    ├─→ 用户点击"委派"按钮（单条或多选后批量）
    │   └─→ 弹出人员选择 Modal
    │       └─→ 选择人员 → 确认
    │           └─→ 资料录入-责任人追加委派人员
    │               └─→ Toast 提示"委派成功，已通知相关人员"
    │
    └─→ 所有条目满足条件后（已录入 + AI 通过）
        └─→ "提交审核"按钮变为可用（仅角色负责人可见）
            └─→ 点击 → 二次确认 Modal
                └─→ 确认 → 数据扭转到维护审核节点
                    └─→ 成功提示 → 返回详情页
```

### 4.2 审核流程（全链路）

```
维护审核负责人进入审核页
    │
    ├─→ 系统加载已提交审核的条目数据
    │
    ├─→ 逐条审核（查看交付件、录入内容、AI 检查结果）
    │   │
    │   ├─→ 单条点击"通过" → 该条维护审核状态变为"通过"
    │   │
    │   └─→ 单条点击"不通过" → 该条维护审核状态变为"不通过"
    │
    ├─→ 点击固定栏"通过"按钮（整体通过）
    │   │
    │   └─→ 弹出确认 Modal："是否需要创建遗留任务？"
    │       │
    │       ├─→ 选"否" → 直接通过
    │       │   └─→ 审核通过 → 成功提示 → 返回详情页
    │       │
    │       └─→ 选"是" → 弹出创建遗留任务 Modal
    │           └─→ 填写遗留任务（至少 1 条）
    │               └─→ 确定 → 审核通过 → 返回详情页
    │
    └─→ 点击固定栏"不通过"按钮（整体不通过）
        │
        └─→ 弹出创建 Block 任务 Modal
            └─→ 填写评审意见（必填）
                └─→ 添加 Block 任务（至少 1 条）
                    └─→ 确定 → 回退到资料录入节点
                        └─→ 飞书通知（Toast 模拟）
                            └─→ 返回详情页
```

### 4.3 关闭流水线流程

```
用户点击列表中的"关闭流水线"
    │
    ├─→ 场景一：无角色进入维护审核（仅申请人可操作）
    │   │
    │   └─→ 弹出关闭流水线 Modal
    │       └─→ 关闭原因文本框（必填）
    │           └─→ 确定 → 流水线关闭
    │               ├─→ 进行中的子流程状态变为"取消"
    │               ├─→ 列表备注列显示取消原因
    │               └─→ 详情页顶部显示取消提示条
    │
    └─→ 场景二：有角色已进入维护审核（仅 SQA 可操作）
        │
        └─→ 弹出关闭流水线 Modal
            ├─→ 各角色评审状态表格（只读）
            │   │ 评审角色 │ 责任人 │ 评审结论 │ 评审意见 │
            │   │ SPM     │ 张三   │ PASS    │ N/A     │
            │   │ 测试    │ 李四   │ Fail    │ xxx     │
            │   │ ...     │ ...    │ ...     │ ...     │
            │
            └─→ 关闭原因文本框（必填）
                └─→ 确定 → 流水线关闭（同上）
```
